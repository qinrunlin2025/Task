### 2.1GitHub是一个面向开源的软件项目的托管平台。

> 开源：源代码。别人引用他人的代码时不违规的，但是应该要标明原作者。
>
> 托管平台：也就是说，GitHub是一个放代码让别人看的地方。

书上说，GitHub是考验一个程序员能否写出源代码的重要标准。

> 全世界的程序员都可以在这看到大佬敲的代码，
>
> 有不会的可以在这里找到解决方案。

> 程序员已经完成的作品可以放心地存在仓库里。

### 2.2Pull request

> 功能：将你的代码变更合并到目标分支（如main/master）

> 创建PR的完整流程
>
> 1. 本地准备
>    1. git checkout -b feature/login #创建新分支
>    2. git add . #添加修改
>    3. git commit -m"添加登录功能“
>    4. git push origin feature/login #推送到远程
> 2. 在GitHub/GitLab 创建PR
>    1. 进入仓库页面→Pull Requests标签页
>    2. 点击New Pull Request
>    3. 选择：
>       + base：main（目标分支）
>       + compare：feature/login（你的分支）
>    4. 填写标题1和描述
>    5. 点击Create Pull Request

### 2.3Fork

> 本义：叉子。引申义：分叉。
>
>  **Git/GitHub仓库克隆**
>
> *相当于你在原项目的主分支上又建立了一个分支，*
>
> *你可以在该分支上任意修改，*
>
> *如果想将你的修改合并到原项目中时，可以pull request，*
>
> *这样原项目的作者就可以将你修改的东西合并到原项目的主分支上去。*
